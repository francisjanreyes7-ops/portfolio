Identity Access Management Certificate Advisor

A six-tier Claude-powered agentic architecture for enterprise certificate services. Turns vague user intent ("my server needs a cert") into the right PKI workflow across EJBCA, public SSL, code signing, S/MIME, and certificate-based authentication — with the right form pre-filled and the right team paged.

Claude API Agentic AI EJBCA PKI ACME / SCEP / EST Azure AD Active Directory
User Intent→ Claude Orchestrator→ Domain Specialist→ CSSP / Team Queue
ProblemCertificate operations span dozens of services (internal PKI, public SSL, code signing, S/MIME, automation), each with different protocols and approvers. Users routinely open tickets to the wrong team or get stuck in protocol-mismatch loops.
ApproachDesigned a six-tier agentic architecture — Enterprise → Super → Domain → Process → Specialist → Utility — where Claude resolves identity, triages intent, detects protocols, and routes to the specialist agent loaded on demand. Backed by a structured Digital Brain of required fields, business rules, and team queues per domain.
Outcome~40% reduction in complex task handling — Claude resolves protocol selection, identity binding, and CSSP form pre-fill autonomously instead of bouncing requests between three teams. First-ask routing accuracy improved from ~50% to ~85%, and average time-to-resolution dropped by an estimated ~60% for routine certificate requests. Operators inherit a system that scales: new domain specialists plug into the same orchestrator pattern without re-architecting the front door.
